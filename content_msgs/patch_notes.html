  • Full-Fledged Settings & Persistence:
    • An "Options" tab with multiple sub-tabs allows for deep customization of nearly every feature.

  • History and Favorites:
    • History: A Treeview log of your recent transformations that can be selected to instantly reload the input, output, and mode. This history can be saved on exit.
    • Favorites: A Listbox where you can save your most-used modes for quick access.
  • Advanced Contextual Controls: The UI is now dynamic, showing specialized controls only when relevant:
    • Morse Code: Audio playback (Play/Save Audio) and sliders to fine-tune dot/dash/space timing.

    • Generators: Controls for Lorem Ipsum (count, unit) and Number Base (from/to base).

    • Modern UI: The entire UI was rebuilt with modern ttk widgets, PanedWindows, and Notebooks, providing a clean, resizable, and professional interface that is far superior to the old static grid of buttons.
    • Bidirectional Live Update: The "Live Transform" feature is now bidirectional. Typing in the "Input" box updates the "Output" box, and typing in the "Output" box will perform a reverse transformation to update the "Input" box.

    • Informative Tooltips: Detailed tooltips have been added to nearly every button and control, explaining its function.
  • Workflow: The fundamental workflow has changed from direct manipulation of the main editor's text to a more robust copy-in, transform, copy-out model, facilitated by the new "Copy on Transform" option and dedicated Copy button.
Fixed
  • Modularity and Code Health: Fixed the poor practice of having a large, unrelated feature tightly coupled within the main application class. The new modular design is significantly cleaner and more maintainable.
  • Lack of User Feedback: The old tool failed silently. The new version provides clear error messages in the output box for invalid transformations (e.g., decoding invalid Base64, incorrect binary input).
  • Scalability: The original design was not scalable. Adding a fifth mode would have required significant UI and logic changes. The new design can accommodate dozens of modes with minimal effort.
  • UI Rigidity: The old window was a fixed size with a static layout. The new window is fully resizable, with panes that can be adjusted by the user to suit their needs.

Search functions (new standalone file)

Added
  • "Smart" Mode: A new "Smart" category that uses a "frecency" algorithm to rank functions based on a weighted score of how frequently and recently they have been used. This list adapts to your personal workflow over time.
  • Function Pinning: Users can now pin their most-used functions. Pinned items are marked with a star (★) and always appear at the top of search results and in their own dedicated "Pinned" category.
  • Contextual Suggestions: When the search box is empty, the list now intelligently suggests actions based on the current application state (e.g., "Copy" and "Cut" are suggested if text is selected; "Save" is suggested if the file has unsaved changes).
  • Advanced Search Syntax: The search input now acts like a command-line interface with special prefixes:
    • ^=query: Filters for functions that start with the query.
    • *=query: Filters for functions that contain the query.
    • mode:category: Switches the search category directly from the input field (e.g., mode:tools).
  • Shortcut Search: You can now find a function by typing its keyboard shortcut (e.g., typing ctrl+s will bring up the "Save" function).
  • Docstring Viewer: A new read-only text box was added to the bottom of the window. It automatically displays the documentation (__doc__) for the currently selected function, explaining what it does.
  • Enhanced UI & UX:
    • A right-click context menu allows for pinning/unpinning functions and clearing history.
    • A clear button (×) was added next to the search entry for quick resets.
    • A hint label was added to inform users about the new advanced search syntax and shortcuts.
    • A results count label in the footer shows how many functions match the current filter.
    • The "All" mode now uses clear category separators (e.g., --- EDIT ---) to group functions visually.
  • Full State Persistence: The tool now saves the following to the EgonTE_settings.json file upon closing:
    • The last search query and selected mode.
    • The list of all pinned functions.
    • The "Smart" mode's history of function usage (MRU list).
Changed/Improved
  • Architecture: The feature was completely extracted from a large, monolithic method within the main Window class into its own open_search_functions function in a separate file. This greatly improves code organization, maintainability, and separation of concerns.
  • Search Algorithm: The original simple substring search (if entry_content in term) was replaced with a multi-layered system:
    1. It first checks for exact shortcut matches.
    2. It then applies the user's chosen strategy (fuzzy, prefix, or contains).
    3. The default fuzzy scoring algorithm is much more powerful, ranking results by relevance and giving bonuses for matches at the start of words.
  • Performance:
    • Debouncing: The search no longer filters on every single keystroke. Instead, it waits for a brief pause in typing (120ms) before updating the results, preventing UI lag and creating a much smoother experience.
    • Caching: The comprehensive map of all functions and their shortcuts is now built once and cached, speeding up subsequent searches.
  • Function Discovery: The new implementation uses a robust ensure_command_sources helper to dynamically and reliably gather all function mappings from the main application, replacing the less reliable dictionary merging of the old version.
  • Keyboard Navigation: Navigation is significantly improved. The Up and Down arrow keys now work from within the search entry field to select items in the list, and Alt+Up/Down can be used to cycle through the search modes without using the mouse.

Fixed

  • UI Responsiveness: The old implementation could become sluggish and unresponsive when typing quickly in the search box. The new debounced filtering completely resolves this issue.
  • List Population Logic: The original version had commented-out code and known issues with correctly displaying all functions, especially in "all" and "others" modes. The new structured list-building process fixes this, ensuring all functions are displayed correctly and in a more organized fashion.
  • State Management: The old version was stateless. The new version's persistence ensures a consistent and personalized experience across application restarts.
